# 问题分析与修复

## 发现的问题

测试运行结果显示：
- 100根K线只生成了1笔
- 中枢数量为0

## 原因分析

### 1. 笔的穿越中轴判断过于严格

当前代码中，笔必须穿越前一笔的中轴：

```python
if bi_list:
    prev_bi = bi_list[-1]
    mid_line = (prev_bi.high + prev_bi.low) / 2
    
    if direction == Direction.UP:
        # 向上笔必须穿越前一笔中轴
        if end_price <= mid_line:
            continue
    else:
        # 向下笔必须穿越前一笔中轴
        if end_price >= mid_line:
            continue
```

**问题**：这个条件在笔的初期阶段会导致大部分笔被过滤掉。

### 2. 解决方案

**方案A：移除穿越中轴判断（简化版）**

对于第一笔，不应该有穿越中轴的要求。只有形成中枢后，后续的笔才需要穿越中轴。

**方案B：放宽穿越条件**

只在形成中枢时才要求穿越中轴，单笔阶段不要求。

## 修复代码
